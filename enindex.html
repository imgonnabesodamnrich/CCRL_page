Paper Notes: Continual Contrastive Reinforcement Learning (CCRL)

Environment-aware fault diagnosis of aero-engines under highly imbalance scenarios

Paper Title: Continual contrastive reinforcement learning: Towards stronger agent for environment-aware fault diagnosis of aero-engines through long-term optimization under highly imbalance scenarios

Journal: Advanced Engineering Informatics (Elsevier), Vol. 65, 2025

DOI: 10.1016/j.aei.2025.103297

1. Background and Problem

Aero-engines are core components of aircraft, and their reliability is crucial. In actual operation and fault diagnosis, engineers face three main challenges:

  1. Highly Imbalanced Samples: Engine design is very precise, operating normally most of the time, making fault data (especially specific types) very scarce. Traditional deep learning models require large and balanced samples for training. Facing "many normal samples, few fault samples," recognition accuracy often drops significantly.
  2. Dynamic Environment Changes: Flight environments, climatic conditions, and engine aging are constantly changing. A static model trained on a specific dataset struggles to adapt to future new situations.
  3. Continuous Data Growth: Airlines generate massive amounts of flight data daily, and traditional methods find it difficult to use this new data to update models in real-time.

To address these issues, the paper proposes a method called CCRL (Continual Contrastive Reinforcement Learning). This method aims to build an agent capable of learning continuously like human experts in long-term environmental interactions, adapting to new data, and handling sample imbalance.

2. Core Idea

The method views the fault diagnosis process as an Agent-Environment Interaction process, including the following innovations:

  • Introduction of Environment Interaction: Utilizes the actual operation process of airlines. Flight data serves as "State," and ground expert fault confirmation results serve as "Feedback." The agent continuously optimizes its judgment strategy based on feedback.
  • Contrastive Learning for Data Scarcity: Designing a contrastive learning encoder for the problem of few fault samples. It uses abundant normal samples to learn general features, then fine-tunes with few fault samples to improve discrimination ability for rare faults by pulling similar samples closer and pushing dissimilar ones apart.
  • Reinforcement Learning for Continual Evolution: Uses an improved Dueling Double Deep Q-Network (D3QN) algorithm combined with a reward mechanism designed for imbalanced samples, allowing the model to update continuously in the data stream instead of stopping after one training session.

3. Methodology

The CCRL framework mainly consists of three parts: Environment Construction, Feature Discrimination Module, and Category Identification Module.

3.1 Environment Interaction Mechanism

Under the reinforcement learning framework, the paper abstracts the actual airline operation process into an "Environment":

  • Input (State): Sensor cross-sectional data transmitted by the aircraft via ACARS system (e.g., deviation values of EGT, fuel flow, speed, etc.).
  • Action: The diagnostic conclusion given by the agent based on current data (normal or a specific fault).
  • Reward: If the agent judges correctly (consistent with the post-event expert maintenance report CNR), it receives a positive reward; otherwise, a negative reward.

This mechanism ensures that the model can continuously self-correct and iterate with daily new data streams.

3.2 Feature Discrimination Module (Contrastive Learning)

This is key to solving the "too few samples" problem. Since fault data is scarce, direct classifier training is prone to overfitting. This study takes two steps:

  1. Autoencoder Pre-training: Uses extremely abundant normal samples to train an Autoencoder. The purpose is to let the network learn how to efficiently compress and restore normal engine operation data and master basic data laws.
  2. Contrastive Learning Fine-tuning: Based on the idea of SimCLR, constructs positive and negative sample pairs.
    • Positive Pairs: Different samples of the same fault type.
    • Negative Pairs: Samples of different fault types.
    • Goal: Adjust network parameters so that data points of the same fault cluster together in feature space, while different faults are as far apart as possible.

Through this step, even if a fault category has only a few samples, the network can extract the most distinguishing features.

Fig 2. Schematic of Feature Discrimination Module — Extracting scarce fault features using contrastive learning

3.3 Fault Identification Module (Reinforcement Learning)

After extracting high-quality features, the paper uses the D3QN algorithm for final fault classification decisions.

  • Imbalanced Reward Mechanism: To prevent the model from tending to predict the "majority class" (normal state), a weighted reward function is designed. If the model successfully identifies rare fault types, it receives a much higher reward weight than identifying normal samples. This forces the agent to pay more attention to "rare" fault cases.
  • Dual Network Structure: Adopts a separate structure for Policy Net and Target Net to improve training stability.

4. Experimental Verification

The paper used real CFM56-7B aero-engine datasets for verification.

Table 1: Performance comparison of different methods on real aero-engine datasets (%)

Method F1 Score Precision Recall
D3QN 77.19 ± 3.81 80.71 ± 4.15 76.75 ± 3.88
DS + D3QN (Under-sampling) 68.15 ± 9.26 71.05 ± 9.10 68.00 ± 9.14
OS + D3QN (Over-sampling) 74.38 ± 5.07 81.20 ± 3.66 74.00 ± 4.90
CCRL (Proposed) 84.26 ± 4.62 87.19 ± 4.34 84.00 ± 4.77
  • Data Composition: Includes normal samples and three typical faults: Variable Bleed Valve (VBV) fault, Exhaust Gas Temperature Indication (EGTI) fault, Total Air Temperature (TAT) sensor fault. Fault samples are extremely scarce (e.g., only 14 samples for a certain fault in the training set), belonging to a highly imbalanced scenario.
  • Baselines:
    1. D3QN (Original Reinforcement Learning)
    2. DS+D3QN (Under-sampling + RL)
    3. OS+D3QN (Over-sampling + RL)
  • Experimental Results:
    • F1 Score: CCRL reached 84.26%, significantly outperforming baseline methods (others between 68%-77%).
    • Stability: In multiple repeated experiments, CCRL had the smallest standard deviation, indicating the most stable performance against randomness.
    • Confusion Matrix Analysis: CCRL's false positive and false negative rates on scarce faults (like EGTI) were lower than traditional data augmentation methods.

5. Conclusion

The proposed CCRL method effectively solves the difficulties of data imbalance and dynamic environmental changes in aero-engine fault diagnosis by combining the feature extraction capability of contrastive learning with the continuous decision-making capability of reinforcement learning. It does not need to generate virtual data (like GAN) out of thin air, but fully exploits the value of existing data, providing a new solution for industrial equipment maintenance under small sample conditions.
